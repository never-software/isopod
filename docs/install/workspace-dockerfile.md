# workspace.Dockerfile

The Dockerfile that defines your workspace container image. Every pod runs from this image, so it should contain everything your project needs — language runtimes, system packages, and pre-installed dependencies.

## Location

```
docker.local/workspace.Dockerfile
```

## What to put in it

- **System packages** — build tools, database clients, libraries your code links against
- **Language runtimes** — Ruby, Node.js, Python, Go, etc.
- **Workspace directories** — `mkdir -p /workspace/<repo-name>` for each repo
- **Dependency pre-install** — handled by [cache hooks](cache-hooks.md) via the `__CACHE_HOOK_INSTRUCTIONS__` placeholder
- **Services** — PostgreSQL, Redis, or anything your app needs at runtime
- **code-server** — browser-based VS Code IDE
- **Startup script** — copy `workspace-start.sh` into the image

## Cache hook placeholder

Add this line where you want dependency install instructions to be injected:

```dockerfile
# __CACHE_HOOK_INSTRUCTIONS__
```

At build time, isopod replaces this with `COPY` + `RUN` instructions generated by your [cache hooks](cache-hooks.md). This gives you per-repo, per-technology Docker layer caching.

## Example

```dockerfile
FROM ubuntu:24.04

# System dependencies
RUN apt-get update -qq && apt-get install -y --no-install-recommends \
  git curl rsync build-essential libpq-dev \
  && rm -rf /var/lib/apt/lists/*

# Language runtimes
RUN curl -fsSL https://deb.nodesource.com/setup_22.x | bash - && \
  apt-get install -y nodejs

# Workspace
RUN mkdir -p /workspace
WORKDIR /workspace

# Dependencies (injected by cache hooks)
# __CACHE_HOOK_INSTRUCTIONS__

# Startup
COPY docker.local/workspace-start.sh /usr/local/bin/workspace-start.sh
RUN chmod +x /usr/local/bin/workspace-start.sh
CMD ["/usr/local/bin/workspace-start.sh"]
```

## Tips

- Put dependency lockfile `COPY` instructions before full source code copies — Docker caches layers by content hash, so unchanged lockfiles mean cached installs
- Use `--no-install-recommends` with apt to keep the image lean
- Install services (Postgres, Redis) in the image but start them in `workspace-start.sh`
